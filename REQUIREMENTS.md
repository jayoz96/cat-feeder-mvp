# 需求清单 - 上门喂猫 MVP (Agentic Coding Challenge)

## 🎯 核心目标
构建一个连接铲屎官（用户）与喂猫员（接单者）的极简平台，解决短期离家时的喂猫需求。

## 👥 用户角色
1. **铲屎官 (User)**: 发布喂猫需求。
2. **喂猫员 (Feeder)**: 接单并执行喂猫任务。
3. **管理员 (Admin)**: 简单后台查看数据（可选 MVP）。

## 📱 核心功能模块 (MVP Scope)

### 1. 认证与个人中心 (Auth & Profile)
- [ ] **快速登录**: 支持手机号/邮箱登录（MVP可使用 Mock 或 Supabase Auth）。
- [ ] **角色切换**: 简单的开关切换“我是铲屎官”/“我是喂猫员”。
- [ ] **地址管理**: 添加/编辑家庭住址（喂猫地点）。

### 2. 铲屎官 - 发布需求 (User Flow)
- [ ] **发布订单**:
    - 选择日期范围。
    - 填写猫咪数量、特殊备注（如：只喂干粮、需要铲屎）。
    - 预估费用计算（按次/按天）。
- [ ] **订单列表**: 查看“待接单”、“进行中”、“已完成”的订单。
- [ ] **支付模拟**: 点击“支付”按钮即视为支付成功。

### 3. 喂猫员 - 接单与执行 (Feeder Flow)
- [ ] **接单大厅**: 查看附近的订单列表（MVP可简化为列表展示）。
- [ ] **任务执行流程**:
    1. **签到**: 到达地点打卡。
    2. **服务中**: 上传照片/视频（MVP可使用 Mock 图片上传）。
    3. **反馈**: 填写喂食情况、猫咪状态。
    4. **签退**: 结束服务。

### 4. 评价与结算 (Review & Settlement)
- [ ] **评价**: 订单完成后，双方互评（星级 + 简短评论）。
- [ ] **结算**: 简单的虚拟钱包余额变动展示。

## 🛠️ 非功能性需求
- **响应式设计**: 优先适配移动端 (Mobile First)。
- **性能**: 页面加载速度 < 2s。
- **数据安全**: 用户隐私（地址、联系方式）脱敏显示。
