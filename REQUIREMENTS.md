# 需求清单 - 上门喂猫 MVP (Agentic Coding Challenge)

## 🎯 核心目标
构建一个连接铲屎官（用户）与喂猫员（接单者）的极简平台，解决短期离家时的喂猫需求。

## 👥 用户角色
1. **铲屎官 (User)**: 发布喂猫需求。
2. **喂猫员 (Feeder)**: 接单并执行喂猫任务。
3. **管理员 (Admin)**: 简单后台查看数据（可选 MVP）。

## 📱 核心功能模块 (MVP Scope)

### 1. 认证与个人中心 (Auth & Profile)
- [ ] **快速登录**: 支持手机号/邮箱登录（MVP可使用 Mock 或 Supabase Auth）。
- [x] **角色切换**: 简单的开关切换”我是铲屎官”/”我是喂猫员”。
- [x] **地址管理**: 添加/编辑家庭住址（喂猫地点）。

### 2. 铲屎官 - 发布需求 (User Flow)
- [x] **发布订单**:
    - 选择日期范围。
    - 填写猫咪数量、特殊备注（如：只喂干粮、需要铲屎）。
    - 预估费用计算（按次/按天）。
- [x] **订单列表**: 查看”待接单”、”进行中”、”待审核”的订单。
- [x] **历史订单**: 独立页面查看已支付订单，支持查看完整详情（照片墙、反馈、评分）。
- [x] **待审核提醒**: 进入订单页自动弹窗提醒有待审核订单。
- [x] **删除订单**: 所有状态的订单均可删除。
- [x] **猫咪日记**: 聚合已完成订单的照片墙和反馈，按时间线展示。
- [x] **再来一单**: 历史订单一键复用，自动预填表单。
- [x] **支付模拟**: 点击”支付”按钮即视为支付成功。

### 3. 喂猫员 - 接单与执行 (Feeder Flow)
- [x] **接单大厅**: 查看附近的订单列表，支持按距离/按时间排序切换。
- [x] **距离排序**: 基于 Haversine 公式计算距离，显示"距你 X.Xkm"标签。
- [x] **任务执行流程**:
    1. **签到**: 到达地点打卡。
    2. **照片墙**: 选择 emoji 猫咪状态卡片（最多 6 张）替代图片上传。
    3. **反馈**: 填写喂食情况、猫咪状态。
    4. **签退**: 结束服务。

### 4. 评价与结算 (Review & Settlement)
- [x] **评价**: 订单完成后，星级评分 + 评论。
- [x] **结算**: 模拟支付宝/微信支付。

## 🛠️ 非功能性需求
- **响应式设计**: 优先适配移动端 (Mobile First)。
- **性能**: 页面加载速度 < 2s。
- **数据安全**: 用户隐私（地址、联系方式）脱敏显示。
